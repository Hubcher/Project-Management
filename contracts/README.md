# Contracts

Пакет `contracts` содержит gRPC-контракты и сгенерированный Go-код для взаимодействия между микросервисами системы управления проектами.

## Назначение

Этот модуль является единым источником истины для межсервисных контрактов.  
Здесь хранятся:

- `proto/.proto`- описания всех gRPC API
- `gen/go/`- сгенерированный Go-код для protobuf и gRPC
- правила генерации и линтинга контрактов

Контракты вынесены в отдельный модуль, чтобы:

- отделить API от реализации микросервисов
- упростить переиспользование контрактов между сервисами
- централизованно управлять изменениями API
- поддерживать версионирование контрактов

## Структура

```text
contracts/
  README.md
  Makefile
  proto/
    auth/
    export/
    project/
    report/
    user/
  gen/
    go/
```

### ‍💻 Генерация кода

В текущей директории `project-management/contracts` установить необходимые зависимости:

```powershell
make tools
```

Сгенерировать Go-код из .proto-файлов

```powershell
make protobuf
```

Проверка protobuf-контрактов

```powershell
make protolint
```

Проверка Go-кода:

```powershell
make golint
```

Запустить все проверки:

```powershell
make lint
```

Очистка сгенерированного кода

```powershell
make clean
```
